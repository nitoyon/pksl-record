会話でポケモンスリープの個体情報を登録するためのスキルです。
複数のスクリーンショットを受け取ります。
スクリーンショットがない場合は、"スクリーンショットが添付されていません" と出力して、完了扱いにしてください。

スクリーンショットがある場合は、以下の処理をしてください。

- 最初に「前処理」を実施します。
- 各スクリーンショットについて、1～5の「スクリーンショットごとの処理」を実行してください。
- 最後に「出力」の処理をします。

**重要**: 処理の途中でユーザーへのテキスト出力を一切行わないでください。画像の読み取り結果、日付の決定、コマンド実行の報告などの途中経過は出力せず、すべての処理が完了した後に「出力例」の形式のみで結果を出力してください。

**重要**: 画像の確認はコンテキストの節約のために実施しないでください。添付ファイルの中身の確認は後述の OCR 処理の結果を利用してください。

# 前処理

プロンプトから次の情報を読み取ってください。

- username: ユーザー名
- displayname: ユーザー表示名

プロンプトの先頭行から「ポケモン名」を読み取ってください。複数のスクリーンショットがある場合は、それぞれのポケモン名が渡されます。

次に、スクリーンショットの前処理として、次のコマンドを実行して、画像を OCR 処理します。

`ndl-ocr\.venv\Scripts\python ndl-ocr\src\ocr.py --sourcedir .tmp --output .tmp`

- 処理には1枚あたり10秒程度の時間を要します。スクリプトが終了するまで待機してください。
- 各画像ファイル名はそのままに、拡張子 .xml のファイルに OCR 結果が出力されます。
  - (例) cdd55385-b416-4946-b1f8-31fa47a20867.png → cdd55385-b416-4946-b1f8-31fa47a20867.xml

# スクリーンショットごとの処理

各スクリーンショットごとに `xml` ファイルを参照して、以下の1～5の処理を実行します。

## 1. 画像サイズの特定

XMLファイルの先頭 2 行目の `PAGE` タグから画像のサイズを読み取ります。

(例) `<PAGE IMAGENAME="cdd55385-b416-4946-b1f8-31fa47a20867.png" WIDTH="1183" HEIGHT="2109">` の場合、1183×2109。

## 2. レベル・ニックネームの読み取り

画像の左上にある「Lv.61〇〇」の文字からレベルとニックネームを読み取ってください。

- (例) `<LINE TYPE="本文" X="255" Y="277" WIDTH="276" HEIGHT="30" CONF="0.929" PRED_CHAR_CNT="3.000" ORDER="16" STRING="Lv.61クワガノン" />` の場合、レベルは「61」、ニックネームは「クワガノン」。
- Lv.ではなくLy.と読み取ることがあるので注意

## 3. スキル確率の読み取り

「詳細ステータス」の上に5つの四角が次のような順番で並んでいます。

```
[Lv.10のサブスキル  ] [Lv.25のサブスキル ]
[Lv.50のサブスキル  ] [Lv.75のサブスキル ]
[Lv.100のサブスキル ]
```

(例) Lv.10: 食材確率アップM、Lv.25: スキル確率アップS、Lv.50 最大所持数アップL、Lv. 75: おてつだいスピードM、Lv.100: 食材確率アップS の場合

必要なレベルに満たないサブスキルは無効です。
2.で取得したレベルに応じて、有効なサブスキルを決定してください。

- 例: レベルが14の場合は、Lv.10のサブスキルのみ有効になります。
- 例: レベルが50の場合は、Lv.10、Lv.25、Lv.50のサブスキルが有効になります。

有効なサブスキルの中に「スキル確率アップM」および「スキル確率アップS」があるかどうかを確認してください。

- 例: スキル確率アップMがある場合
  - スキル確率アップMの有無: 1
  - スキル確率アップSの有無: 0
- 例: スキル確率アップM、Sの両方がある場合
  - スキル確率アップMの有無: 1
  - スキル確率アップSの有無: 1
- 例: スキル確率アップM、Sのどちらもない場合
  - スキル確率アップMの有無: 0
  - スキル確率アップSの有無: 0


## 4. せいかくの変化量の読み取り

「詳細ステータス」のすぐ下に「せいかく」の項目があります。
「せいかく」の右側には、せいかくによるステータスの変化量が記載されています。

- 「せいかくによる特徴なし」と記載されている場合は、せいかくの変化量はすべて0としてください。
- 特徴がある場合は順番に「上昇の項目」「下降の項目」が記載されます。
  - 上昇の項目に「メインスキル発生確率」がある場合は1としてください。
  - 下降の項目に「メインスキル発生確率」がある場合は-1としてください。
  - それ以外の場合は、0としてください。

## 5. スクリプトの実行

以下のコマンドを実行してスプレッドシートに記録してください。

```
venv\Scripts\python iv_update.py ユーザーID ユーザー表示名 ニックネーム ポケモン名 スキル確率アップMの有無 スキル確率アップSの有無 せいかくの変化量
```

例: `venv\Scripts\python iv_update.py nitoyon2 nitoyon ピカチュウくん ピカチュウ 1 0 1`

ユーザー表示名やニックネームに空白を含む場合は、コマンドライン引数のパースにエラーが起きないようにクオートしてください。

コマンドに失敗したときは出力結果とともに、そのことをユーザーに伝えてください
読み取れなかった場合は、読み取れなかった項目をユーザーに伝えてください。

# 出力

すべてのスクリーンショットについて記録が終わったら、以下の内容を `!discord send <<EOF` 形式で送信してください。

```
📝nitoyonさんのポケモンを記録しました

<:biscuit:1488499182648954891>ピカチュウくん
* **種族名**: ピカチュウ
* **スキ確**: M⬜ S✅
* **せいかく**: スキル▲▲

<:biscuit:1488499182648954891>スキ1
* **種族名**: ライコウ
* **スキ確**: M⬜ S⬜
* **せいかく**: スキル補正なし
```

- 1 行目には「<ユーザー表示名>さんのポケモンを記録しました」としてください。
  - <ユーザー表示名> は、プロンプトから読み取ったユーザー表示名に置き換えます。
- 2 行目以降には、スクリーンショットごとに以下のフォーマットで記録内容を記載してください。複数のポケモンがいる場合は、間に空行をあけてください。
  - フォーマット: `<:biscuit:1488499182648954891><ニックネーム>` `種族: <ポケモン名>` `スキ確: M[スキル確率アップMの有無] S[スキル確率アップSの有無]` `せいかく: [せいかくの変化量])` を1行ずつ
  - <ニックネーム> は 2 で読み取ったニックネームを記載する
  - <ポケモン名> は前処理で読み取った値を記載する
  - [スキル確率アップM] は 3 で読み取ったスキル確率アップMの有無を表す
    - あるとき: ✅
    - ないとき: ⬜
  - [スキル確率アップS] は 3 で読み取ったスキル確率アップSの有無を表す
    - あるとき: ✅
    - ないとき: ⬜
  - [せいかくの変化量] は 4 で読み取ったせいかくの変化量を表す
    - 0: スキル補正なし
    - 1: スキル▲▲
    - -1: スキル▼▼
