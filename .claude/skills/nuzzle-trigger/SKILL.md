会話でスキルの記録をリクエストされたときの処理を定めています。

添付ファイルがなく、発言者のポケモン名修正を依頼された場合は、末尾の回復対象置換処理を実行してください。それ以外の場合は完了扱いにしてください。

添付ファイルがある場合は、添付ファイルごとに以下の処理をしてください。

- 最初に「前処理」を実施します。
- 各スクリーンショットについて、1～3の「スクリーンショットごとの処理」を実行してください。
- 最後に「出力」の処理をします。

**重要**: 処理の途中でユーザーへのテキスト出力を一切行わないでください。画像の読み取り結果、日付の決定、コマンド実行の報告などの途中経過は出力せず、すべての処理が完了した後に「出力例」の形式のみで結果を出力してください。

**重要**: 画像の確認はコンテキストの節約のために実施しないでください。添付ファイルの中身の確認は後述の OCR 処理の結果を利用してください。

# 前処理

プロンプトから次の情報を読み取ってください。

- username: ユーザー名
- displayname: ユーザー表示名
- channelname: チャンネル名
- created: 日時 (ISO 8601形式)

スクリーンショットの前処理として、次のコマンドを実行して、画像を OCR 処理します。

`ndl-ocr\.venv\Scripts\python ndl-ocr\src\ocr.py --sourcedir .tmp --output .tmp`

- 処理には1枚あたり10秒程度の時間を要します。スクリプトが終了するまで待機してください。
- 各画像ファイル名はそのままに、拡張子 .txt のファイルに OCR 結果が出力されます。
  - (例) cdd55385-b416-4946-b1f8-31fa47a20867.png → cdd55385-b416-4946-b1f8-31fa47a20867.txt

# スクリーンショットごとの処理

各スクリーンショットごとに `txt` ファイルを参照して、以下の1～3の処理を実行します。

## 1. スキル回数の決定

- 「ほっぺすりすり(げんきエールS)」という文字がスキルの発動を表します。
- スキルは2回発動することがあります。この場合、下のスキルを1回目、上のスキルを2回目とします。
- スキルが存在しない場合にはユーザーにエラーを通知してください。

## 2. スキルレベル・スキル発動者・回復対象・追加ボーナス有無の決定

スキルの発動ごとに、それぞれのスキル発動のあとの文字から「スキルレベル」「スキル発動者」「回復対象」「追加ボーナス有無」を読み取ってください

- 「ほっぺすりすり(げんきエールS)」の右に「Lv.6」のように記載されているスキルレベル(1-6)を読み取ってください。
- スキル名の下に「〇〇のメインスキル!」と記載されています。〇〇が「スキル発動者」です。
   - 例: 「■デマルさんのメインスキル！」と表示された場合、スキル発動者は「■デマルさん」です。
- その下に「〇〇のげんきが××回復！」と表示されています。〇〇が「回復対象」です。
   - 例: 「おてきののげんきが35回復！」と表示された場合、回復対象は「おてきの」です。 (「の」の重複に注意)
   - 例: 「みつのげんきが35回復！」と表示された場合、回復対象は「みつ」です。
- 「さらに、〇〇はメインスキルの準備が整った！」というテキストの有無で「追加ボーナス有無」を判定してください。
   - テキストがある場合: 1 (ボーナスあり)
   - テキストがない場合: 0 (ボーナスなし)

## 3. スクリプトの実行

スキルの発動ごとに、以下のコマンドを実行してスプレッドシートに記録してください。
1枚スクリーンショットに2回のスキル発動がある場合、下のスキル(1回目)から先に記録してください。

```
venv\Scripts\python nuzzle_recorder.py 日時 チャンネル名 ユーザー名 ユーザー表示名 発動者 スキルレベル 回復対象 追加ボーナスの有無
```

例: `venv\Scripts\python nuzzle_recorder.py 2024-01-01T12:00:00.000Z チャンネル名 nitoyon2 nitoyon 温丸 6 おてぼんね 1`

ユーザー表示名やニックネームに空白を含む場合は、コマンドライン引数のパースにエラーが起きないようにクオートしてください。

コマンドに失敗したときは出力結果とともに、そのことをユーザーに伝えてください

# 出力

すべてのスクリーンショットについて記録が終わったら、以下の内容を `!discord send <<EOF` 形式で送信してください。

```
📝nitoyonさんのすりすりを記録しました
* <:energy:1488038051832791132>温丸 (Lv.6) <:bonus:1488040085499809792> おてぼんね
* <:energy:1488038051832791132>温丸 (Lv.6) <:nobonus:1488040146250240041> おてぼんね
```

- 1枚のスクリーンショットに複数のスキル記録がある場合は、その分だけ行を追加してください。
- 複数のスクリーンショットがある場合は、その分だけ行を追加してください。
- 1行目には `📝<ユーザー表示名>さんのすりすりを記録しました` と出力します。
- 2行目以降には、スキル発動ごとに以下の内容を出力してください。
  - `<:energy:1488038051832791132><発動者> (Lv.<スキルレベル>) <追加ボーナスの有無><回復対象>`
  - `<追加ボーナスの有無>` は、以下のように表現してください。
    - 追加ボーナスあり: `<:bonus:1488040085499809792> `
    - 追加ボーナスなし: `<:nobonus:1488040146250240041> `

その後 `.tmp` フォルダに存在している `*.txt` `*.json` `*.xml` ファイルを削除してください。

# 回復対象置換処理

プロンプトからユーザー名を読み取ってください。

- username: ユーザー名

依頼された置換処理に応じて、指定されたユーザー名のポケモン名の置換スクリプトを読んでください。

例: `venv\Scripts\python ユーザーID 旧ポケモン名 新ポケモン名`
