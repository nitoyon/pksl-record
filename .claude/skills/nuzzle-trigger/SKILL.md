name: nuzzle-trigger

---

会話でスキルの記録をリクエストされたときの処理を定めています。
スクリーンショットがない場合は、"スクリーンショットが添付されていません" と出力して、完了扱いにしてください。

スクリーンショットがある場合は、以下の処理をしてください。

- 最初にプロンプトからユーザー情報を読み取ります。
- 各スクリーンショットについて、1～5の「スクリーンショットごとの処理」を実行してください。
- 最後に「出力」の処理をしてください。

# ユーザー情報の読み取り

プロンプトから次の情報を読み取ってください。

- username: ユーザー名
- displayname: ユーザー表示名
- channelname: チャンネル名
- created: 日時 (ISO 8601形式)

# スクリーンショットごとの処理

## 1. スキル回数の決定

- 下側の白い四角にスキル発動を示すテキストからが記載してあります。
- スキルは2回発動することがあります。この場合、下のスキルを1回目、上のスキルを2回目とします。
- スキルの種類を読み取ってください
  - スキルの種類が「ほっぺすりすり (げんきエールS)」ではない場合は、ユーザーにエラーを通知してください。

## 2. スキルレベル・スキル発動者・回復対象・追加ボーナス有無の決定

スキルの発動ごとに、それぞれのスキル発動の四角の中から「回復対象」「追加ボーナス有無」を読み取ってください

- スキル名の右の「スキルレベル」(1-6)を読み取ってください
- スキル名の下に「〇〇のメインスキル！」と表示されています。〇〇が「スキル発動者」です。
- その下に「〇〇のげんきが××回復！」と表示されています。〇〇が「回復対象」です。
- 「さらに、〇〇はメインスキルの準備が整った！」というテキストの有無で「追加ボーナス有無」を判定してください。
   - テキストがある場合: 1 (ボーナスあり)
   - テキストがない場合: 0 (ボーナスなし)

## 3. スクリプトの実行

スキルの発動ごとに、以下のコマンドを実行してスプレッドシートに記録してください。

```
venv\Scripts\python nuzzle_recorder.py 日時 チャンネル名 ユーザー名 ユーザー表示名 発動者 スキルレベル 回復対象 追加ボーナスの有無
```

例: `venv\Scripts\python nuzzle_recorder.py 2024-01-01T12:00:00.000Z チャンネル名 nitoyon2 nitoyon 温丸 6 おてぼんね 1`

ユーザー表示名やニックネームに空白を含む場合は、コマンドライン引数のパースにエラーが起きないようにクオートしてください。

コマンドに失敗したときは出力結果とともに、そのことをユーザーに伝えてください

# 出力

すべてのスクリーンショットについて記録が終わったら、以下の内容を `!discord send <<EOF` 形式で送信してください。

```
📝nitoyonさんのすりすりを記録しました
* <:energy:1488038051832791132>温丸 (Lv.6) <:bonus:1488040085499809792> おてぼんね
* <:energy:1488038051832791132>温丸 (Lv.6) :nobonus:1488040146250240041おてぼんね
```

- 1枚のスクリーンショットに複数のスキル記録がある場合は、その分だけ行を追加してください。
- 複数のスクリーンショットがある場合は、その分だけ行を追加してください。
- 1行目には `📝<ユーザー表示名>さんのすりすりを記録しました` と出力します。
- 2行目以降には、スキル発動ごとに以下の内容を出力してください。
  - `<:energy:1488038051832791132><発動者> (Lv.<スキルレベル>) <追加ボーナスの有無><回復対象>`
  - `<追加ボーナスの有無>` は、以下のように表現してください。
    - 追加ボーナスあり: `<:bonus:1488040085499809792> `
    - 追加ボーナスなし: `<:nobonus:1488040146250240041> `
