name: cheer-trigger
description: げんきエール記録
---

会話でスキルの記録をリクエストされたときの処理を定めています。
6枚のスクリーンショットを受け取ります。6枚でない場合はエラー終了してください。

## 1. スキル発動画像の位置を特定

- 6枚のうち、最初の1枚、もしくは、最後の1枚のスクリーンショットがスキル発動時のものです。どちらがスキル発動時のスクリーンショットかを判断してください。
   - スキル発動のスクリーンショットには「〇〇のメインスキル！」のようなテキストが下部に表示されます。
   - それ以外の5枚は、ステータスを表すスクリーンショットです。左上に SP が表示されているのが特徴です。
- スキル発動が **1枚目** の場合: ステータス画像はスキル発動**後**の状態 → げんきの**減算が必要**
- スキル発動が **6枚目** の場合: ステータス画像はスキル発動**前**の状態 → げんきの**減算不要**

## 2. 日付の決定

スキル発動のスクリーンショットについて、次の処理を実施します。

- プロンプトで日付を指定されたときには、その日付を利用してください。
   - 日付は曜日や昨日・今日・明日などの表現で指定されることもあります。
- 未指定のときは以下の手順で決定してください。
   0. スクリーンショットの左上に日付が記載してある場合はその日付を利用する。
   1. `date` コマンドを実行して現在の日時（日付と時刻）を取得する。
   2. スクリーンショットから時刻を読み取る（例: 23:11）。
   3. 手順1で取得した当日の日付とスクリーンショットの時刻を組み合わせた日時が、現在日時よりも未来かどうかを判定する。
      - 例: 現在が 2/17 0:37 で、スクリーンショットの時刻が 23:11 の場合、2/17 23:11 は未来になる。
   4. 未来になる場合は、前日の日付を採用する（上記の例では 2/16）。未来でなければ当日の日付を採用する。
- 日付はすべてYYYY-MM-DDの形式に変換してください。

## 3. スキルの情報を取得

スキル発動のスクリーンショットについて、次の処理を実施します。

- 下側の四角のスキル発動を示すテキストから、スキルの種類を読み取ってください
  - スキルの種類は「げんきエールS」「マイナス (食材ゲットS)」「ほっぺすりすり (げんきエールS)」のいずれかです
  - それぞれ、「げんきエール」「マイナス」「ほっぺすりすり」に変換してスキル名としてください。
- スキル名の右の「スキルレベル」(1-7)を読み取ってください
- 「回復対象」と「げんき回復量」を読み取ってください
  - スキル名の下に「〇〇のげんきが××回復！」と表示されています。
  - 〇〇が「回復対象」で、「××が回復量」です
- スキルは2回発動することがあります。この場合、下のスキルを1回目、上のスキルを2回目とします。回復量と回復対象が異なることがあります。

## 4. ポケモンごとのげんきの取得

ステータスの5枚のスクリーンショットは、それぞれ1匹目〜5匹目のポケモンに対応しています（スクリーンショットの順番がそのままポケモンの番号です）。それぞれについて、次の処理を実施します。

- ポケモンイラストの左下にある「げんき」の値 (0%～150%)を読み取ってください
   - 例: 128% → 128
- ポケモンイラストの下にある「名前」を読み取ってください

## 5. 回復対象のポケモンが何匹目かの特定とげんきの減算

3.で取得した「回復対象」の名前と、4.で取得した各ポケモンの名前を照合して、回復対象が何匹目（1〜5）かを特定してください。

1. の処理でスキル発動が **1枚目** だった場合（減算が必要）、さらに次の処理をしてください。

- 回復対象のポケモンのげんきの値から、回復量を減算する
  - 例: げんきが 128 で回復量が 20 の場合、128 - 20 = 108

スキルが2回発動している場合は、そのそれぞれのスキルに対して上記の処理を行う

## 6. スクリプトの実行

以下のコマンドを実行してスプレッドシートに記録してください。
スキルが2回発動している場合は、そのそれぞれについて、次の処理を実施してください。
5. でげんきを減算している場合は、減算後のげんきをスクリプトに渡してください。

```
venv\Scripts\python cheer_recorder.py YYYY-MM-DD hh:mm 1匹目のげんき 2匹目のげんき 3匹目のげんき 4匹目のげんき 5匹目のげんき スキルの種類 スキルレベル 対象ポケモンが何匹目か 回復量
```

例: `venv\Scripts\python cheer_recorder.py 2026-02-01 14:30 128 100 95 85 75 げんきエール 3 1 20`

コマンドに失敗したときは出力結果とともに、そのことをユーザーに伝えてください
読み取れなかった場合は、読み取れなかった項目をユーザーに伝えてください。

### 出力例

以下の内容を複数行メッセージとして送信してください。

```
📝記録しました (スキル発動1枚目)
* 2月1日 14:30 げんきエール (128 / 100 / 98 → 118 / 85 / 75)
```

- `日時 スキル名 (各ポケモンのげんきの値)` を表記する
- かっこの中には各ポケモンのげんき量を記載する
- 回復対象のポケモンについては `回復前のげんき量 → 回復後のげんき量` を記載する
- 回復後のげんきの値が150を超える場合は、150として表示すること
